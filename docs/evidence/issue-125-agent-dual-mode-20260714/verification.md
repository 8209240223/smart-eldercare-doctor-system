# Issue #125 验证记录

## 1、修复范围

1. 助手新增 `自动`、`普通问答`、`站内 Agent` 三种模式。
2. 普通问答模式不向 Kimi 提供任何站内工具，避免知识问答误查数据库。
3. Agent 能力目录补充路径参数、查询参数、请求体字段及基础校验信息。
4. 能力目录上限由 80 扩展到可覆盖当前全部业务映射，并新增全路由自动审计测试。
5. 修复医生被 Agent 外层白名单错误阻止读取护理计划、护理记录及相关统计的问题，最终权限仍由控制器 `@RequireRole` 与后端拦截器判断。
6. 站内请求在首次模型调用前预检当前角色可用接口，命中后直接调用业务接口，减少一次工具发现往返。

## 2、自动化验证

- 后端全量测试：187 个通过，0 个失败。
- Agent 聚焦测试：17 个通过，覆盖双模式、参数目录、角色隔离、路由覆盖与内部调用。
- 前端 lint：通过；仅保留仓库原有的两个 Fast Refresh 警告。
- 前端生产构建：通过。
- Spring Boot 可执行包：构建成功。

## 3、真实浏览器验证

Playwright 在本地 `8080` 上完成以下只读流程，未执行新增、编辑或删除操作：

1. 普通问答模式完成高血压饮食知识问答，Agent 步骤数量为 0。
2. 站内 Agent 查询当前医生可见的老人档案总数，显示工具调用步骤并返回真实数量。
3. 医生通过 Agent 查询后端授权的护理计划数据，没有出现 403 或越权提示。
4. 移动端 390×844 视口中，对话框和三段模式切换均未溢出。

本轮实测耗时：普通问答 3171 ms，老人档案查询 4169 ms，护理计划查询 4156 ms。完整机器结果见 `result.json`。
