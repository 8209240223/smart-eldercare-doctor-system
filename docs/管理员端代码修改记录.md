## 二、后端改动明细

### 改动 1：新建注解 @RequireRole

**文件**：[RequireRole.java](file:///c:/Users/Lenovo/Desktop/smart-eldercare-doctor-system/src/main/java/com/medical/common/annotation/RequireRole.java)

**关键说明**：

```java
@Target({ElementType.TYPE, ElementType.METHOD})  // 同时支持类和方法
@Retention(RetentionPolicy.RUNTIME)              // 运行时保留，便于反射读取
public @interface RequireRole {
    int[] value();                                // 1=管理员 2=医生 3=护士
}
```

**设计要点**：

- **类级 + 方法级**：`@RequireRole` 可贴在 Controller 类上（整类生效）或方法上（方法覆盖类），便于"类宽松 + 方法收窄"的常见模式。
- **运行时保留**：拦截器通过反射读注解，必须 `RUNTIME`。
- **int[] 数组**：支持多角色（如 `{1,2,3}` 表示三端通用）。

**回归验证**：所有 Controller 反射读取正常，`@RequireRoleTest` 3 个用例通过。

### 改动 2：新建拦截器 RoleInterceptor

**文件**：[RoleInterceptor.java](file:///c:/Users/Lenovo/Desktop/smart-eldercare-doctor-system/src/main/java/com/medical/common/interceptor/RoleInterceptor.java)

**关键说明**：

```java
// 必须注册在 JwtInterceptor 之后
// JwtInterceptor.order(1) → 注入 currentUserType
// RoleInterceptor.order(2) → 读取 currentUserType 校验
```

**设计要点**：

1. **顺序严格**：注册时 `order(2)` 必须在 JWT 之后，否则 `currentUserType` 为 null 会导致所有接口 401。
2. **OPTIONS 放行**：CORS 预检请求不带 token，必须放行。
3. **非 HandlerMethod 放行**：静态资源、错误页等不进拦截逻辑。
4. **方法级优先**：先读方法注解，没有再读类注解，没有则放行（保持旧行为）。
5. **错误响应统一**：通过 `R.fail(401, "未登录或Token无效")` / `R.fail(403, "当前角色无权访问该功能")` 返回，前端可按 code 统一处理。

**回归验证**：`RoleInterceptorTest` 10 个用例全部通过，覆盖：
- 无注解放行
- 类级注解 / 方法级注解
- 管理员允许、医生/护士拒绝
- 无 currentUserType → 401
- OPTIONS 放行
- 非 HandlerMethod 放行
- 多角色 {1,2,3} 全部允许
- 非法 userType 格式 → 401
- 错误响应 JSON 内容

### 改动 3：WebMvcConfig 注册双拦截器

**文件**：[WebMvcConfig.java](file:///c:/Users/Lenovo/Desktop/smart-eldercare-doctor-system/src/main/java/com/medical/common/config/WebMvcConfig.java)

**关键说明**：

```java
registry.addInterceptor(jwtInterceptor)
        .addPathPatterns("/api/**")
        .excludePathPatterns(excludes)
        .order(1);
registry.addInterceptor(roleInterceptor)
        .addPathPatterns("/api/**")
        .excludePathPatterns(excludes)
        .order(2);
```

**设计要点**：

- **排除名单与原配置完全一致**：登录、注册、验证码、密码重置、SSE 流、静态资源。任何漏项都会导致对应功能 401/403。
- **顺序不可调换**：若 `roleInterceptor.order(1)`，则拦截时 JWT 还没注入 `currentUserType`，导致所有带注解的接口都返回 401。

**回归验证**：登录、登出、改密码、老人档案查询、警告 SSE 流均不受影响。

@GetMapping("/todo")
public R<?> getTodoList(HttpServletRequest request) {
    Long currentUserId = (Long) request.getAttribute("currentUserId");
    Integer currentUserType = (Integer) request.getAttribute("currentUserType");
    Long todoUserId = currentUserId;
    if (todoUserId == null) todoUserId = 2L;

**文件**：[app.js](file:///c:/Users/Lenovo/Desktop/smart-eldercare-doctor-system/src/main/resources/static/pages/app.js)

**设计要点**：

- **医生专属写**（预警、随访、老人档案、清理计划）：用 `!isAdmin && !isNurse`（等价于"仅医生"），确保管理员、护士都看不到。

- **合并规则**：若原按钮已有 `v-if`，用 `&&` 追加，不覆盖原条件。

**回归验证**：
- 管理员：所有业务页只见列表和"查看"，写按钮消失。
- 医生：所有写按钮可见可用。
- 护士：护理/体征可写，预警/随访/老人档案不可写。
